Software Design and Engineering Enhancement

Artifact Overview

Artifact: Raspberry Pi Thermostat Enhancement

Category: Software Design and Engineering

Summary: This artifact demonstrates how I improved the software design of a Raspberry Pi thermostat application by refactoring a single-file implementation into a modular, maintainable system.

Enhancement Description

In the original version of the thermostat project, the application logic was concentrated in one file. While the system functioned, this structure made the code harder to maintain, test, and expand. Multiple responsibilities were handled together, including sensor reading, display updates, LED control, and serial communication.

In the enhanced version, I redesigned the application into a modular architecture. I separated major responsibilities into dedicated components, including sensor management, LED control, display handling, serial communication, and thermostat control logic. This created a clearer structure and improved the overall organization of the codebase.

This enhancement made the system more maintainable and more scalable. Each module now has a more focused purpose, making future improvements easier to implement without disrupting unrelated parts of the system.

Narrative

For the software design and engineering component of my ePortfolio, I enhanced my original thermostat artifact by improving its architecture and organization. The original implementation was functional, but it relied on a more monolithic design in which several responsibilities were combined in one program. This made the code more difficult to read, maintain, and extend.

In the enhanced version, I applied software design principles to separate the system into multiple modules with clearly defined responsibilities. I created dedicated files for sensor reading, display management, LED control, serial communication, and thermostat control. This modular structure improved separation of concerns and made the application easier to understand and manage.

This redesign also improved maintainability because changes to one part of the system could be made with less risk of affecting the entire application. For example, updates to the display behavior or sensor processing could be handled in their own modules without rewriting the rest of the system. This is a stronger engineering approach because it supports reuse, testing, and future expansion.

Overall, this enhancement demonstrates my ability to evaluate an existing software system, identify design weaknesses, and refactor it into a more structured and maintainable solution. It reflects my growth in software design, modular programming, and professional development practices throughout the Computer Science program.
