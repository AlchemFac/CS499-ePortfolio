Software Design and Engineering Artifact Overview

The original thermostat system was implemented as a single-file program where sensor input, display handling, LED control, and system logic were tightly coupled. This structure made the code difficult to maintain, debug, and extend as additional features were introduced.

To address these limitations, the system was refactored into a modular architecture consisting of dedicated components such as sensor_reader, thermostat_controller, display_manager, and serial_manager. Each module is responsible for a specific aspect of the system, allowing for improved separation of concerns and clearer system organization.

This enhancement demonstrates key software engineering principles including modularity, abstraction, and maintainability. The redesigned system is more scalable, easier to troubleshoot, and better aligned with professional development practices.

Algorithms and Data Structures Artifact Overview

The original implementation relied on individual sensor readings to control system behavior, which introduced instability due to natural fluctuations in temperature and humidity data. This resulted in inconsistent responses from the thermostat.

The enhanced system incorporates a sliding window approach using a deque data structure to store recent sensor readings. This allows the system to calculate moving averages and analyze trends over time rather than reacting to isolated data points. Additionally, rate-of-change calculations were introduced to improve decision-making based on environmental changes.

These improvements produce more stable and reliable system behavior by reducing sensitivity to noise in sensor data. This enhancement demonstrates the practical application of data structures and algorithmic techniques to improve real-world system performance.

Database Artifact Overview

The original thermostat system processed temperature and humidity data in real time but did not retain any historical data. This limited the system’s ability to analyze trends or evaluate long-term performance.

The enhanced system introduces persistent data storage by recording temperature, humidity, and timestamp values. This allows environmental data to be stored and retrieved over time, enabling long-term monitoring and future analysis.

By incorporating a database component, the system evolves from a real-time controller into a data-driven application. This enhancement demonstrates an understanding of data persistence, organization, and the importance of storing system data for analysis, debugging, and performance evaluation.
