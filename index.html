Software Design and Engineering Artifact Overview

This enhancement focuses on improving the software architecture of the Raspberry Pi thermostat project. The original implementation was reorganized from a more monolithic structure into a modular design with dedicated components for sensor input, display handling, LED control, serial communication, and system logic.

These changes improved maintainability, separation of concerns, and the overall quality of the application. The redesign better reflects software engineering principles used in professional development environments.

Database Artifact Overview

This enhancement focuses on improving the thermostat project by adding database functionality to store sensor readings over time. The original implementation handled live temperature and humidity data, but it did not preserve those readings for future use.

By adding a database management component, the system became more effective at organizing and retaining environmental data. This improvement supports persistence, long-term monitoring, and future analysis of thermostat performance.
